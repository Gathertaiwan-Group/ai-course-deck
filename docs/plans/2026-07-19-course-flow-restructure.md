# 課程流程重整執行計畫

> **給執行者：** 依序完成每項工作，並在每次修改後執行對應驗證。

## 1. 建立內容驗收測試

- 將投影片數更新為 19 張。
- 將每張的標題與安全關鍵字寫入 `tests/content.test.mjs`。
- 先執行 `npm test`，確認舊版投影片因不符合新規格而失敗。

## 2. 重整投影片 HTML

- 以已核准的五步網站工作流替換現有 16 張內容。
- 保留標準語意、data label 與可存取性標記。
- 將安全流程拆為曝光、Vercel、GitHub、local 四張可行動投影片。

## 3. 補齊視覺與文件

- 優先重用現有的 chapter、卡片、流程與安全樣式；只新增必要樣式。
- 更新 README 的 19 張投影片清單與安全說明。

## 4. 驗證與發布

- 執行 `npm test` 與 HTML 驗證。
- 以本機靜態伺服器檢查桌機、平板與手機版面。
- 合併至 `main`、推送 GitHub，確認 Vercel Production 部署完成。
