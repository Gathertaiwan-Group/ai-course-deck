AI WEB COURSE

用 AI 打造全端網站

從一句想法，到真正上線。

AI Studio × GitHub × AI Coding × Vercel × Supabase

AI 幫你把想法做成網站。

TODAY’S OUTCOME

今天做出可被找到的網站

  1. 01能被使用有完整的前端與資料
  2. 02能被分享有公開網址與網域
  3. 03能被找到有 GA 與 Search Console

做出能用、能分享、能被找到的網站。

BEFORE CLASS

前置準備

三個帳號、兩組憑證，一台電腦。

  • 開好 GitHub 帳號
  • 開好 Vercel 帳號，取得 Vercel token
  • 開好 Supabase 帳號，取得 Supabase API key

先準備帳號和 key。

TODAY'S WORKFLOW

5 步完成網站

  1. 01生成前端
  2. 02存進 GitHub
  3. 03本地開發
  4. 04部署完成
  5. 05持續優化

從想法開始，一步一步完成真正上線的網站。

每個工具各做一件事。

STEP 011-1

AI STUDIO

AI Studio：快速做出前端

先把需求講清楚，再讓 AI 產出第一版。

  1. 01誰會使用這個網站
  2. 02使用者要完成什麼
  3. 03需要哪些畫面與區塊

說清楚需求，AI 幫你做畫面。

STEP 011-2

前端完成前，先檢查 3 件事

畫面好看之外，也要讓分享與搜尋正常運作。

  1. Meta 資訊
    title、description
  2. Open Graph
    分享預覽圖與文字
  3. RWD
    手機、平板、桌機

確認手機、分享和搜尋都正常。

STEP 022-1

SAVE THE FRONTEND

從 AI Studio 存到 GitHub

用 AI Studio 的匯出功能，把前端放進自己的 GitHub repository。

AI Studio→GitHub repo→第一版

把第一版存到 GitHub。

STEP 033-1

LOCAL FIRST

本地端專案資料夾

在自己的電腦建立專案資料夾，再把 repository clone 下來。

  1. 01建立一個清楚命名的資料夾
  2. 02clone GitHub repository
  3. 03用本機預覽確認前端

在電腦保留專案資料夾。

STEP 033-2

AI CODING

Claude Code 或 Codex 連動 repo

在本地端打開專案，讓 AI 理解程式後協助你修改。

  • Claude Code讀專案、修改、測試
  • Codex新增功能、檢查、提交

讓 AI 先看懂，再幫你改。

STEP 044-1

DEPLOY

Vercel 部署前端

匯入 GitHub repository，取得第一個公開網址。

  1. 01Import Git Repository
  2. 02設定 Environment Variables
  3. 03部署並測試網址

用 Vercel 把網站公開。

STEP 044-2

DATA + ITERATION

Supabase 串資料，持續修改到完成

先接上需要的資料，再反覆修改、測試、推送。

修改→測試→push

串上資料，持續修改。

STEP 044-3

AI MAINTENANCE

讓 Claude Code 或 Codex 協助維護

請它先讀專案規則；每次只完成一個可驗證的改動。

  1. 01描述需求與驗收方式
  2. 02修改後先測試、檢查
  3. 03確認後 commit，再 push

一次改一件事，先測試。

STEP 044-4

設定自動更新（推送即部署）

本地改完後，只要 push，Vercel 會自動更新線上網站。

自動部署設定成功後，記得把 GitHub repository 設為 Private。

改好送出去，線上網站就會自己變新。

STEP 055-1

網站完成後的三件事

  1. 01接上 Domain使用自己的網址
  2. 02加入 GA了解使用者行為
  3. 03設定 Search Console讓 Google 能收錄

接網域、GA 和 Search Console。

STEP 055-2

AI + BROWSER

網站上線後，用 AI 處理安全設定

開啟 Claude on Chrome 或 Codex on Chrome，讓 AI 操控瀏覽器；我們只要看著流程即可。

  1. 01換掉舊 key

    重新產生曾貼給 AI 的 key，更新 Vercel；有使用自動化才更新 GitHub Actions Secrets。

  2. 02確認網站安全

    確認 repository 設為 Private，重新部署並測試網站。

不要把密碼、token 和 private key 交給 AI；真正輸入時由自己手動處理。

上線後，請 AI 用瀏覽器把安全設定做完。

  1. 01生成前端

  4. 04部署完成
