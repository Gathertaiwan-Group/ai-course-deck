THE WORKFLOW

05

5 步完成網站

生成前端 → 存進 GitHub → 本地開發 → 部署完成 → 追蹤成效

STEP 0101

AI STUDIO

AI Studio：快速做出前端

先把需求講清楚，再讓 AI 產出第一版。

  1. 01誰會使用這個網站
  2. 02使用者要完成什麼
  3. 03需要哪些畫面與區塊
STEP 0101

前端完成前，先檢查 3 件事

畫面好看之外，也要讓分享與搜尋正常運作。

  1. Meta 資訊
    title、description
  2. Open Graph
    分享預覽圖與文字
  3. RWD
    手機、平板、桌機
  4. 逐頁預覽
    一次只改一件事
STEP 0202

SAVE THE FRONTEND

從 AI Studio 存到 GitHub

用 AI Studio 的匯出功能，把前端放進自己的 GitHub repository。

AI Studio→GitHub repo→第一版
STEP 0303

LOCAL FIRST

本地端專案資料夾

在自己的電腦建立專案資料夾，再把 repository clone 下來。

  1. 01建立一個清楚命名的資料夾
  2. 02clone GitHub repository
  3. 03用本機預覽確認前端
STEP 0303

AI CODING

Claude Code 或 Codex 連動 repo

在本地端打開專案，讓 AI 理解程式後協助你修改。

  • Claude Code讀專案、修改、測試
  • Codex新增功能、檢查、提交
STEP 0404

DEPLOY

Vercel 部署前端

匯入 GitHub repository，取得第一個公開網址。

  1. 01Import Git Repository
  2. 02設定 Environment Variables
  3. 03部署並測試網址
STEP 0404

DATA + ITERATION

Supabase 串資料，持續修改到完成

先接上需要的資料，再反覆修改、測試、推送。

修改→測試→push
SECURITY!

ROTATE BEFORE LAUNCH

貼給 AI 的 key，視為已曝光

所有貼給 AI 的 token 與 API key，都要在正式上線前重新產生。

  1. 01可以公開的只有必要資訊

    Supabase anon key 必須搭配 RLS。

  2. 02私密 key 不進前端

    private key、service-role key、Vercel token 絕不可放進 frontend。

SECURITY01

新 key：手動更新 Vercel

到 Project Settings → Environment Variables，逐一替換新 key。

  1. 01刪除舊值，貼上新值

    確認 Preview 與 Production 使用正確環境。

  2. 02重新部署

    部署完成後，以網站功能實測確認。

SECURITY02

有自動化才更新 GitHub Secrets

只有 GitHub Actions 自動化流程有使用時，才需要更新 GitHub Actions Secrets。

  1. 01Repository Settings

    進入 Secrets and variables → Actions。

  2. 02名稱不變，值換新

    更新後重新執行 workflow 並確認成功。

SECURITY03

同步本地 .env.local，測試後再推送

本機 `.env.local` 也換成新 key；不要 commit 私密值到 GitHub。

  1. 01本機啟動測試

    確認讀取到的是新設定，而不是舊快取。

  2. 02檢查 .gitignore

    `.env.local` 必須被排除，推送前再檢查一次。

MAINTAIN06

CLAUDE CODE

讓 Claude Code 協助維護

請它先讀專案規則；每次只完成一個可驗證的改動。

  1. 01描述需求與驗收方式
  2. 02修改後先測試、檢查
  3. 03確認後 commit，再 push
WHEN STUCK↗

Chrome 上的 AI 助手

設定卡住時，讓 AI 直接看著畫面、協助操控瀏覽器。

  1. Claude on Chrome

    協助閱讀頁面、點選設定與完成重複操作。

  2. Codex on Chrome

    協助操控瀏覽器、檢查網頁與排除卡住的步驟。

只在設定卡住時使用；不要輸入敏感資料、token 或 private key。

AUTOMATION→

推送即部署

讓 GitHub 連動 Vercel；Claude Code 不需要拿到 Vercel token。

自動部署設定成功後，記得把 GitHub repository 設為 Private。
